VScode 重装日记

背景

上个月想换一个背景，使用了background插件，发现效果有点不好。对比之下VS背景设置就很不错。
卸载了插件后，还是一直在弹VScode有问题的弹窗，建议直接重新安装。但是我前几周太忙，而且一直在学python，所以说没有在意这个。
问题出现在昨天，准备写一个C语言的程序，发现直接运行不了。猜测是在我前几天配置pytorch时，腾内存时，把C语言的配置文件之类的东西给删了。
所以说，想从头开始。卸载了VScode和配置文件。
连C盘里的.VScode和code文件都删了

.VScode储存配置文件，code则是用户信息。

  • 配置C语言环境
  • 配置MD预览，毕竟用VScode很多时间是用来记笔记的。

原教程

1. 为Windows安装C编译器（MinGW-W64 GCC）

MinGW-W64 GCC下载地址
下载到了D盘，再加入到环境变量：
![](VScode配置日记\屏幕截图 2022-11-03 232715.png)

cmd.exe窗口中，输入gcc -v命令，会出现gcc的版本号，说明安装配置成功。

2. 配置VScode

  • 安装C/C++扩展，安装Code Runner扩展。
  • 俩扩展安装完成后，是对“Code Runner”扩展的配置，在“文件”菜单下找到“首选项”再找到“设置”子菜单。将“Run Code Configuration”中的“Run In Terminal”、”Save File Before Run”前面的复选框选中。

    目测是设置了：在终端中run，而不是控制台。如果之后要在控制台中画贪吃蛇之类的，可以改回来。
    没配置中文插件的话，名字可能有差。

3. 开发C语言工程

出人意料的是，我没有自己配置很多文件，反而是只增加了两个文件，还是自动配置的。
我的步骤：

  1. 创建了一个C语言程序
  2. 在终端页面点击配置任务，自动创建了.VScode文件夹，里面有一个task.json。
  3. 这就可以运行了，调试也没问题

VScode 美化

说到这个，就不得不夸VScode了。
VScode主打的就是一个轻量化，而且有巨多的扩展。

  1. Bracket Pair Colorizer –> 给匹配的括号着色
  2. Indent-Rainbow 让缩进带有颜色
  3. Path Intellisense 自动完成文件名
  4. Power Mode —> 什么生产监督器，不过效果还是不错的
  5. TODO Highlight
  6. TA-super-transla 划词翻译。鼠标停留选中几秒就会查看翻译内容
  7. filesize 在状态栏中显示当前文件大小
  8. VS Code Counter 统计代码总行数、注释行数、空白行数，以及使用的语言。使用方法：右键需要统计的文件夹—Count lines in directory
  9. Rainbow Fart 官方概述：VSCode Rainbow Fart是一个在你编程时持续夸你写的牛逼的扩展，可以根据代码关键字播放贴近代码意义的真人语音。
  10. Error Lens
